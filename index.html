Quem é o Eixo de Avaliação

Esta página apresenta a composição do time responsável pela gestão, produção, revisão, organização e acompanhamento das avaliações. A proposta é facilitar o contato de pessoas de fora do eixo e deixar mais claro o papel de cada pessoa na operação.

Quando o proxy do Notion enviar os dados da equipe, esta aba será preenchida automaticamente. Enquanto isso, o HTML mantém uma versão-base com espaço para foto, Slack e e-mail.

Como este painel lê o Notion

Este HTML é somente leitura. Ele não cria, edita nem apaga nada no Notion. A base compartilhada continua funcionando normalmente para as outras áreas.

Filtro padrão aplicado pelo HTML: mostra apenas itens com Eixo = Avaliação e Arquivar Projeto ≠ marcado.

Campos usados no database “(Planejamento) Projetos”

  • Nome → avaliação/entrega macro.
  • Prazo → início previsto e entrega prevista. Se o database usar outro nome, o HTML também tenta ler: Data, Período, Entrega, Data de entrega ou Entrega prevista.
  • Status → status macro.
  • Responsáveis → responsável macro.
  • Descrição do Projeto → observação executiva.
  • URL → link do entregável ou referência.
  • Tarefas Relacionadas → usado apenas para tentar descobrir a entrega real, sem mostrar microetapas.

Automação com Notion via Netlify

Este pacote já inclui o arquivo netlify/functions/notion.js. O token fica seguro nas variáveis de ambiente do Netlify.

export default async function handler(req, res) {
  res.setHeader("Access-Control-Allow-Origin", "*");
  res.setHeader("Access-Control-Allow-Methods", "GET,OPTIONS");
  res.setHeader("Access-Control-Allow-Headers", "Content-Type");
  if (req.method === "OPTIONS") return res.status(200).end();

  const token = process.env.NOTION_TOKEN;
  const databaseId = process.env.NOTION_DATABASE_ID;

  const notionHeaders = {
    "Authorization": `Bearer ${token}`,
    "Notion-Version": "2022-06-28",
    "Content-Type": "application/json"
  };

  let allResults = [];
  let cursor = undefined;

  do {
    const response = await fetch(`https://api.notion.com/v1/databases/${databaseId}/query`, {
      method: "POST",
      headers: notionHeaders,
      body: JSON.stringify({
        page_size: 100,
        start_cursor: cursor,
        filter: {
          and: [
            { property: "Arquivar Projeto", checkbox: { equals: false } },
            { property: "Eixo", multi_select: { contains: "Avaliação" } }
          ]
        }
      })
    });

    const data = await response.json();
    if (!response.ok) return res.status(response.status).json(data);

    allResults = allResults.concat(data.results || []);
    cursor = data.has_more ? data.next_cursor : undefined;
  } while (cursor);

  for (const page of allResults) {
    const rel = page.properties?.["Tarefas Relacionadas"]?.relation || [];
    page.relatedTasks = [];

    for (const item of rel) {
      const taskResp = await fetch(`https://api.notion.com/v1/pages/${item.id}`, {
        headers: notionHeaders
      });
      const task = await taskResp.json();
      page.relatedTasks.push(task);
    }
  }

  res.status(200).json({ results: allResults, lastSync: new Date().toISOString() });
}

Regras dos Pontos de Atenção

O bloco “Pontos de atenção” é calculado automaticamente pelo HTML. Ele considera:

  • Avaliações com progresso abaixo de 30%.
  • Risco de conflito entre cronogramas, quando há sobreposição de períodos.
  • Entregas previstas para a mesma semana.
  • Ausência de data de início ou entrega prevista.
  • Avaliações pausadas.
  • Prazos vencidos ou próximos, quando faltam até 5 dias.

Campos sugeridos no Notion para a aba “Eixo de Avaliação”

Para preencher a equipe automaticamente, o proxy precisa enviar uma lista chamada equipe. O HTML está preparado para ler os seguintes campos:

  • Nome → nome da pessoa.
  • Cargo ou Função → papel no eixo.
  • Descrição, O que faz ou Responsabilidades → resumo da atuação.
  • Slack → usuário ou link do Slack.
  • E-mail ou Email → contato por e-mail.
  • Foto ou Imagem → arquivo ou URL da foto.
